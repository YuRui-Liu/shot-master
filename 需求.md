# 分镜提示词反推 Web App · 需求规格

**项目代号**：shot-prompt-backwards
**版本**：v1.0 (2026-05-12)
**状态**：需求评审中（待用户审阅）

---

## 1. 项目目标

构建一个**本地 Web App**，把分镜图（拼接宫格/独立多图/单图/整个文件夹）通过 vision 模型反推成 ComfyUI LTX 2.3 视频生成工作流可用的**结构化提示词**（global_prompt / timeline_data / segment_lengths / max_frames / frame_indices / strengths / epsilon / notes）。

同时整合 `shot-master` 项目的**网格拆分 / 网格组合 / 去白边**能力作为 Web 端工具页，避免在两个工具间频繁切换。

**核心价值**：现有提示词反推靠手动复制图片到 AI 客户端 + 复制提示词模板 + 复制结果回 ComfyUI，每镜要 5-8 次切换；本工具把全流程压缩到"选图 → 选模板 → 一键反推 → 一键复制 / 保存"。

---

## 2. 用户与场景

**用户**：项目作者本人（短剧/动画分镜创作者，使用 ComfyUI 生成 LTX 2.3 视频）。

**典型场景**：
- **A. 单镜临时反推**：手头一张 1×3 三联图，想立刻得到首中尾三帧的提示词，复制到 ComfyUI 跑视频
- **B. 一集批量反推**：某集导出了 20 张分镜宫格图，想一次性反推完所有提示词，落盘为 md+json，后续逐镜送 ComfyUI
- **C. 拆图工作流**：原 16 宫格大图 → 拆成 4 张 4 宫格子图 → 落盘 → 后续手工选其中 1-2 张做反推

## 3. 范围

### 3.1 In Scope（v1.0）

**反推核心**：
- 4 种输入模式：拼接宫格图 / 多张独立图（按顺序选） / 单张普通图 / 整个文件夹（批量模式）
- 多 vision 后端可切换：Gemini / OpenAI 兼容（含 OpenAI / DeepSeek / OpenRouter / 字节豆包 Ark / 本地 vLLM / SiliconFlow 等所有 OpenAI 兼容端点） / Anthropic Claude / Qwen-VL（DashScope）
- 反推模板可增删改：基于现有 `prompts/分镜提示词反推/` 下 3 套 md 模板（三帧首中尾 / 四帧 / 多帧自适应），用户可在 Web 内增删改
- 模板自动推荐 + 手动覆盖
- 结构化补充输入表单（剧本、台词、fps、总时长、风格备注、角色资料等），表单字段由模板 frontmatter 定义
- 批量模式下补充输入支持两种复用策略：全部复用同一份 / 按文件名规则逐项映射（同名 .md/.json 自动绑定）
- 反推结果：md 文件保存 + json 文件保存 + Web UI 展示且可编辑后重保存 + 一键复制各字段

**拆图（防灾门禁）**：
- 宫格模式下，反推前必须先拆图预览
- 预览 N 张子图缩略图，用户须勾选"我确认拆图正确"才能解锁"开始反推"
- 文件夹批量模式：批量预览所有拆图缩略图，整体人工确认后再批量反推
- 任何一张拆图错误可剔除或调整网格规格重拆

**整合 shot-master 功能（独立 Tab）**：
- **拆图 Tab**：R×C 网格 → 子网格拆分，落盘为独立子图（复用 `shot_master.core.splitter`）
- **拼图 Tab**：N 张图 → 拼接为 R×C 网格图（复用 `shot_master.core.combiner`）
- **去白边 Tab**：单图或批量裁白边（复用 `shot_master.core.border_detector`）

**配置管理**：
- API key 和默认后端配置：`.env` 文件 + Web UI 设置页双通道
- 设置页可查看、修改、运行时切换当前后端
- 历史管理：每次反推生成的 md/json 直接落盘到指定输出目录，不维护额外历史数据库

### 3.2 Out of Scope（不做）

- 不做用户系统、多用户、权限管理
- 不做云端部署（仅本地 localhost）
- 不做 ComfyUI 节点封装（已在另一个赛道，本项目专注 Web App 形态）
- 不做反推结果的 SQLite 历史库（v1.0 后视需要再加）
- 不做拆图/拼图/去白边的复杂可视化交互（保留 PySide6 桌面版作为高级版）
- 不做手机/平板适配（PC 桌面浏览器即可）

## 4. 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 后端 | FastAPI + uvicorn | 与现有 Python 脚本同栈；原生 SSE 支持；异步友好 |
| 前端 | 原生 HTML + Alpine.js + fetch | 零构建链；本地工具不需要 React 复杂度；调试简单 |
| 图像核心 | 复用 `shot-master` 包 | 已有完整 splitter/combiner/border_detector + 测试 |
| 配置 | python-dotenv + settings.json | .env 存敏感信息；settings.json 存运行时偏好 |
| 多后端 SDK | google-genai / openai / anthropic / dashscope | 各家官方 SDK |

**整合 shot-master 策略**：
- `pyproject.toml` 把 `shot-master` 加为本地可编辑依赖：`pip install -e ../shot-master`
- 不复制源码，直接 `from shot_master.core import split_image, combine_images, GridSpec`
- shot-master 的 PySide6 桌面版继续保留，互不冲突
- 启动脚本检查依赖，未安装时自动 install

### 5.2 Web UI Tab 规划

| Tab | 功能 | 实现 |
|---|---|---|
| **反推** | 主功能：图 → 提示词 | 新建（带防灾门禁） |
| **拆图** | R×C 网格 → 子网格切分，落盘 | 复用 `shot_master.core.splitter` |
| **拼图** | N 图 → 网格拼接 | 复用 `shot_master.core.combiner` |
| **去白边** | 单图/批量裁白边 | 复用 `shot_master.core.border_detector` |
| **模板** | 反推模板 md 增删改 | 新建 |
| **设置** | API key / 默认后端 / 默认模型 | 新建 |

### 5.3 反推 Tab 流程

```
1. 选输入模式：拼接宫格 / 多图 / 单图 / 文件夹批量
2. 选图：缩略图列表
3. （仅宫格模式）拆图预览 → 缩略图 → 用户勾选「我确认拆图正确」
4. 选反推模板（自动推荐 + 手动覆盖）
5. 填补充输入（基于模板 frontmatter 自动生成表单）
6. 开始反推（宫格模式下被 step 3 勾选门禁锁）
7. 结果可编辑 → 保存 md+json + 一键复制
```

**防灾规则**（强约束）：
- 宫格模式下，拆图预览必须先跑且必须勾选确认才能进入反推
- 不做"记住选择"，每次都要重新确认
- 批量模式：先批量预览拆图 → 整体确认 → 再批量反推；任何一张错误可剔除/换规格

### 5.4 数据流

**路径 A · 单次反推**（同步）：
```
POST /api/inference {images:[paths], template_id, supplement, override?}
→ grid_split(if needed) → provider.generate() → output_writer.write_md_json()
← {global_prompt, timeline_data, ..., md_path, json_path}
```

**路径 B · 批量反推**（异步 + SSE）：
```
POST /api/batch {folder, template_id, supplement_strategy}
← {task_id}
GET /api/batch/{task_id}/stream  (Server-Sent Events)
事件序列:
  progress  {idx:3/20, current_file, status:'running'}
  item_done {idx:3/20, md_path, json_path, status:'ok'}
  item_done {idx:4/20, error:'...', status:'failed'}
  complete  {ok:18, failed:2, output_dir}
```

**路径 C · 拆图预览**（同步本地）：
```
POST /api/grid/preview {image_path, src_grid, sub_grid}
→ shot_master.core.splitter.split_image() → 临时存 app/.cache/preview/{hash}/
← {tiles:[url, ...]}     // url 形如 /static/cache/preview/{hash}/tile_0.png
```
`.cache/preview/` 目录在每次启动时清空，避免磁盘膨胀。

## 6. 关键组件

### 6.1 VisionProvider 抽象

```python
class VisionProvider(ABC):
    @abstractmethod
    def generate(self,
                 images: list[Path],
                 system_prompt: str,
                 user_supplement: str
                 ) -> str:
        """返回 vision 模型原始文本输出"""

    @classmethod
    @abstractmethod
    def available_models(cls) -> list[str]:
        """该后端支持的模型名列表"""
```

**实现清单**：
- `gemini.py` — google-genai SDK
- `openai_compat.py` — 一类多用，支持任意 OpenAI 兼容端点（base_url + api_key + model 三参定位）。覆盖 OpenAI 官方、DeepSeek、OpenRouter、字节豆包 Ark、本地 vLLM、SiliconFlow 等
- `anthropic.py` — anthropic SDK
- `qwen_vl.py` — DashScope SDK

**provider 工厂**：注册表模式，加新后端类 = 新建文件 + 注册一行。对于 OpenAI 兼容的新厂商，**只需在 `factory.py` 加一个 endpoint 预设**（base_url + 默认模型名），不用新建文件。

### 6.2 模板系统

**模板文件格式**（YAML frontmatter + 正文）：

```markdown
---
name: 多帧自适应
suggest_when: image_count >= 4
variables:
  - {name: total_seconds, type: int, default: 16, label: 总时长(秒), required: true}
  - {name: fps, type: int, default: 24, label: FPS, required: true}
  - {name: key_frame_count, type: int, default: 4, label: 关键帧数, required: true}
  - {name: style_note, type: text, label: 风格备注, optional: true}
  - {name: script, type: textarea, label: 剧本/台词, optional: true}
---
你是 LTX-Video 2.3 多帧引导提示词工程师...
（正文可用 {{total_seconds}} {{fps}} {{script}} 占位符）
```

**变量类型支持**：`int` / `float` / `text` / `textarea` / `select`（带 `options`）/ `file_pick`（从项目文件夹挑 .md/.json/.txt 自动导入内容）

**自动推荐规则**：
- `image_count == 1` → 单帧模板
- `image_count == 3` → 三帧首中尾
- `image_count == 4` → 四帧
- `image_count >= 5` → 多帧自适应
- 始终允许下拉框覆盖

### 6.3 task_runner（批量执行）

- **执行模式**：串行（一张一张做完再下一张）
- **进度通信**：SSE 推送 `progress` / `item_done` / `complete` 事件
- **失败处理**：单条失败不中断后续；失败项可在前端列表里单独点重试
- **断点续跑**：v1.0 不做（输出目录已有同名文件时跳过 = 隐式续跑）

## 7. 错误处理

| 错误类别 | HTTP | 行为 |
|---|---|---|
| 输入校验失败（图片不存在 / 网格规格无效） | 400 | 友好文案 → 浏览器红条 |
| 上游 vision API 错误（限频 / 超时 / key 无效） | 502 | 原始错误信息 → 浏览器提示 + 单条重试按钮 |
| 内部异常 | 500 | 写入 `logs/error.log` + 返回 task_id 让用户上报 |
| 模板渲染错误（占位符未提供） | 400 | 指明缺哪个字段 → 表单高亮 |
| 文件 IO 错误（输出目录不可写） | 500 | 提示用户检查权限/路径 |

**vision API 重试策略**：
- 限频错误（429）：自动 sleep + 重试 1 次，仍失败则报给前端
- 超时（>60s）：直接报错，不重试
- 不做指数退避循环（避免静默卡死）

## 12. 风险与开放问题

| 风险 | 影响 | 应对 |
|---|---|---|
| shot-master 路径变更或不稳定 | 整合失效 | pyproject.toml 用绝对路径或 fallback 到内嵌副本 |
| 各家 vision SDK 升级 break API | 单后端失效 | 锁版本 + 抽象层隔离 + provider 单测 mock |
| 大批量任务网络中断 | 部分结果丢失 | 输出文件落盘即视为安全，重启后跳过已存在的同名文件 |
| 用户误删反推模板 | 流程中断 | 模板 Tab "导入示例" 按钮可恢复 3 套默认模板 |
| 拼接宫格图过大（>20MB） | 上传/上传 base64 超限 | 自动压缩到 2048px 长边（vision 模型也用不上更高分辨率） |

**开放问题**（v1.1 再考虑）：
- 是否加 SQLite 历史库做语义检索？
- 是否加模板版本管理（git-style diff）？
- 是否做 ComfyUI 节点版（与本 Web App 双形态）？
- 是否支持反推结果直接 PATCH 进 ComfyUI 工作流 JSON？
