# 糯米 AI 短剧项目 · 文件结构规范

> 2026-06-01。供其他应用/工具遵循本项目的磁盘文件约定。来源：`drama_shot_master/core/compass/paths.py`、`manifest.py`、`ref_index.py`、各 agent 落盘逻辑。
> 原则：**磁盘产物是唯一真相源**（展示层扫描磁盘判定状态，manifest 仅元信息/记账）。关联 [[gui-web-rewrite-strategy]]、`立意切换归档方案与接口标准.md`。

## 1. 项目根目录布局
项目目录名 = `P-NNN_<slug>`（如 `P-003_叽里咕噜`），位于 `projects_root` 下。
```
P-003_叽里咕噜/
  project.json                 # manifest（元信息+流程记账，见 §2）
  创意.json                    # 立意阶段产物（候选/选定/对话，见 §3）
  剧本.json                    # 剧本索引（分集大纲，见 §4）
  剧本_E1.md  剧本_E2.md …      # 分集正文（每集一文件；旧单文件 剧本.md = E1）
  分镜_E1.json  分镜_E2.json …  # 分镜脚本（每集一文件；旧 分镜.json = E1，见 §5）
  prompts/                     # 出图提示词产物（见 §6）
    E1/
      角色参考图/  *.md
      N宫格/       *.md
  characters/                  # 角色资源（ref_index + ref 图，见 §7）
    ref_index.json
    <角色名>_ref.png
  scenes/                      # 场景资源（同结构）
    ref_index.json
  props/                       # 道具资源（同结构）
    ref_index.json
  imggen/                      # 出图产物（分镜图）：shot_<shotId>_NN.png（见 §8）
  video/                       # 视频生成产物（LTX）
  video_prompts/               # 视频提示词产物（集级）
  audio_prompts/               # 配音提示词产物（集级）
  soundtrack/                  # 配乐/BGM 产物
  assets/                      # 通用资产
  exports/                     # 成片导出
  归档/                        # 立意切换归档区（见 立意切换归档方案，<idea_id>__<标题>/）
```
路径常量：`MANIFEST_FILENAME=project.json`、`REF_INDEX_FILENAME=ref_index.json`、`RESOURCE_KINDS=(characters,scenes,props)`、archive_dir=`归档`、idea=`创意.json`。

## 2. project.json（manifest）
```json
{
  "schema_version": 1,
  "project_id": "P-003",
  "project_name": "叽里咕噜",
  "genre": "<genre_id 字符串，立意选定后由 PUT /project/meta 写>",
  "params": {},
  "style_bible": { "ref": "<style_id>", "name": "<风格名>" },
  "status": "empty|ideating|scripted|…",
  "pipeline": { "screenwriter": {"state":"pending|in_progress|completed","next_action":""},
                "assets": {...}, "storyboard": {...}, "production": {...} },
  "artifacts": {}, "episodes": {}, "dependencies": {},
  "archive": [ {"idea_id","title","dir","archived_at","files":[]} ],
  "created_at": "ISO8601", "last_modified": "ISO8601"
}
```
- **重要**：manifest 的 `pipeline.*.state` 历史上不随产物更新（僵尸快照）。**消费方应以磁盘产物扫描判定阶段状态**（见 §9），manifest 仅取 project_id/name/genre/style_bible/archive 等元信息。

## 3. 创意.json（立意）
```json
{
  "input": { "core_idea","genre_tags","format","tone_tags","visual_style","candidate_count","duration_sec","extra_constraints","genre_context","style_context" },
  "messages": [ {"role":"user|assistant","content":"…"} ],
  "candidates": [ {"id":"c1","title","angle","summary","highlights","est_duration"} ],
  "selected_id": "<当前选定候选 id，空=未选>",
  "updated_at": "ISO8601"
}
```
- 兼容旧名 `idea.json`（读取两名都接受，写入用 创意.json）。
- 注意：`input` **不含** style_bible/genre 对象（后端 context 模型剥离）；题材/风格的权威存储在 **manifest**（genre/style_bible），由 PUT /project/meta 写。

## 4–5. 剧本 / 分镜
- **剧本索引** `剧本.json`：`{ episodes: [ {id:"E1", title, summary} ], … }`。
- **分集正文** `剧本_E{id}.md`：Markdown 正文。旧单文件 `剧本.md` 视为 E1。
- **分镜** `分镜_E{id}.json`：`{ title, aspectRatio, totalDuration, globalStyle, characters:[{name,appearance}], shots:[{shotId,duration,composition,description,stylePrompt}] }`。旧 `分镜.json` 视为 E1。

## 6. prompts/（出图提示词）
- `prompts/E{id}/角色参考图/*.md`、`prompts/E{id}/N宫格/*.md`（嵌套子目录，非递归列举需逐子目录）。

## 7. 资源库 ref_index（characters/scenes/props）
- 每类一个 `<kind>/ref_index.json`：`{ schema_version, refs: [ {name, path, source, status} ] }`。
  - `name`：稳定引用名（下游 prompt 头部用 `@<name>_ref.png` 引用）。
  - `path`：相对项目/子目录的落盘文件路径。
  - `source`：`template | custom | ai-generated`。
  - `status`：`ready | pending`（ready 可下游消费）。
- ref 图落 `<kind>/<name>_ref.png`。角色 ref 生成应注入项目 style_bible（风格锁）+ 角色三视图无背景约束。

## 8. imggen/（分镜图产物）
- 命名约定：`imggen/shot_<shotId>_NN.<ext>`（NN 两位序号）。
- 消费（分镜视角/视频模式2 重载已生成图）：按 `shot_<shotId>_` 前缀过滤发现，取最新一张预览。

## 9. 阶段状态判定（扫描磁盘，唯一真相源）
| 阶段 | done 判定 |
|---|---|
| ideate | 创意.json 存在且 selected_id 非空（存在未选=进行中） |
| script | 存在 剧本_E*.md 或 剧本.md |
| storyboard | 存在 分镜_E*.json 或 分镜.json |
| imggen/prompts | prompts/ 目录非空 |
| production/video | video/ 目录有文件 |
- `GET /project/overview` 已按此扫描实现。

## 10. 媒体访问 / 端点约定
- 本地媒体经 `GET {media_agent}/file?path=<相对项目或绝对>&project=<dir>` 流式返回（图/视频/音频，按扩展名 content-type）。前端 http 页禁 file://，一律走 /file。
- 列目录 `GET {media_agent}/project/files?project=&sub=&ext=`（非递归）。
- 概览 `GET /project/overview`、元信息写 `PUT /project/meta {project, genre?, style_bible?, params?}`。
- 立意切换归档见 `立意切换归档方案与接口标准.md`（归档/ 目录，永不删除）。
