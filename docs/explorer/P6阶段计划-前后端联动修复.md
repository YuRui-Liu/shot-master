## 一个统一病根：三套"真相源"不一致
P-003 磁盘实况：`创意.json`(2候选/6条messages/**selected_id=""**) + `project.json`(manifest:**style_bible={} / pipeline 全 pending / episodes={} / artifacts={}**,停在创建时刻) + `video/`。
- **真实产物**（screenwriter_agent 直接写盘：创意.json/剧本_E*.md/分镜_E*.json/prompts/）
- **僵尸 manifest**（media_agent project.json，仅创建时写一次，产物产生后从不更新）
- **静态 mockup 页面**（分镜视角 storyboard-board、视频模式2 video-mode2 SHOTS 硬编码）

overview/资源库读 manifest → 永远空；分镜视角/模式2 读硬编码 → 永远静态。**修复主线 = 让展示层以"扫描真实产物"为唯一真相源**（参考 screenwriter_agent `core/project_scanner.py` 已有的扫描逻辑），而非信任 manifest 或静态数据。

## 分项根因 + 修复 + 优先级

### P6-1【数据持久化】概览风格圣经丢失 + 剧本选定丢失 — 🔴最高
**根因**：
- `创意.json` selected_id="" 但有候选+6条messages（续聊其实成功）。`web/ideate.html` advanceBtn 里 `try{ fetch("/ideate/select") }catch{}` **吞掉错误仍照常 postMessage 推进** → select 失败/未落盘也"看起来推进了"，selected_id 没写。
- 风格圣经：ideate 选风格只写 state+localStorage，**没写进 创意.json input.style_bible，也没写进 manifest.style_bible**（input keys 里只有 visual_style/style_context，无 style_bible）→ 概览读 manifest.style_bible={} → 空。
**修复**：
- ideate advanceBtn：检查 `/ideate/select` resp.ok，失败则提示+不推进（不再静默吞错）。
- 持久化风格圣经 + 题材到项目：新增/复用后端写路径（manifest.style_bible/genre + 创意.json input.style_bible），ideate 选定即写；概览读真实值。
- 落实"选定"语义：/ideate/select 成功 → selected_id 落 创意.json（已实现，确保前端真调成功）。

### P6-2【追踪失效】概览 & 资源库对 P-003 零追踪 — 🔴最高
**根因**：`media_agent/routes/projectx.py:219 /project/overview` 用 `load_manifest()` 读僵尸 manifest（pipeline 全 pending、episodes={}），不反映已存在的 创意.json/video/。
**修复**：
- `/project/overview` 改为**扫描真实产物**判定阶段状态（创意.json→ideate done、剧本_E*.md→script、分镜_E*.json→storyboard、prompts/→prompts、video/→production），manifest 仅作辅助/元信息。可直接复用 screenwriter_agent `core/project_scanner.py` 的扫描思路或调其 GET /project。
- 资源库 ref 追踪：同理，ref_index 读真实落盘 + 概览"最近产物"接 /project/clips（已接，确认 sub 目录覆盖真实产物路径）。

### P6-3【分镜板·分镜视角】静态模板，完全没接后端 — 🔴高
**根因**：`web/storyboard-board.html:773` 明示"静态 mockup：按钮均为占位 stub，不接真实后端"，硬编码"城郊废品回收站"等。
**修复**：分镜视角整体接活——读真实 分镜_E{id}.json（/file + /project/files 列集）渲染镜头卡；每镜头"出图"接 /imggen/generate（带参考图/预置提示词片段，复用 P4 已做的能力）；产物用 mediaUrl 显示。**工作量大**（等于把该页从 mockup 变功能页）。

### P6-4【视频模式2】1-1 城郊废品回收站 静态，未与分镜对齐 — 🔴高
**根因**：`web/video-mode2.html:1513 const SHOTS=[...]` 硬编码；shot-select 下拉也是写死三项。虽有 runVideoPrompt(SSE) 但 SHOT 卡渲染自静态 SHOTS。
**修复**：模式2 SHOT 列表改为读真实分镜（/project/files 列 分镜_E*.json + /file 读）→ 按 shot 渲染卡，与分镜对齐；shot-select 下拉填真实集/镜头；"批量提取"已接 /video_prompt，确保填回真实卡。

### P6-5【视频模式1】长度文本框 + 提交时长口径 + 轮询 SSL — 🟡中
- **长度输入框**：模式1 当前段"长度"滑块旁加文本框，可直接输入秒（或帧，由"时间显示"radio 决定），与滑块双向同步。
- **提交时长口径**：提交 body 的总时长 = **各段时长之和**（不含时间轴空隙/空轨），确认 dirBuildBody 用 Σsegment.duration 而非 timeline span。
- **query_task SSL EOF 重试**：`A query_task 连续3次失败: EOF occurred in violation of protocol (_ssl.c:1007)` — RunningHub 轮询遇瞬时 SSL 断连即放弃。修 providers/runninghub.py（或 video/tts submit 的 wait/poll）：query_task 加**指数退避重试**(如 3-5 次，间隔 1/2/4s)，区分瞬时网络错误 vs 真实失败；超时给清晰提示。

### P6-6【分镜板·自由模式】打开目录/清空失效 + 缺刷新 + 出图面板过小 — 🟡中（含需确认 mock）
**根因**：自由模式部分按钮仍是 stub（同 P6-3 的静态 mockup 范畴）；"打开目录"未走 open.path 中继或目标无效；"清空"未实现；无"刷新"重载目录变化。出图面板布局过小失衡。
**修复**：
- 打开目录 → `postMessage({type:'open.path',path})`（壳已接 Qt openPath）；清空 → 清源图/产物列表 state+重渲；新增"刷新"按钮 → 重新 /project/files 列目录重载（应对外部新增图）。
- **出图面板布局** → 先出 HTML mock 让用户浏览器确认（见下），再改。
