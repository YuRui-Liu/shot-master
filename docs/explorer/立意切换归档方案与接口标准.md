# 立意切换「归档而非删除」方案与接口标准

> 2026-06-01。解决：在已生成剧本/分镜后，点[立意]「推进/换立意」会清空后续所有资源（数据丢失）。
> 目标：**换立意时把旧立意的下游产物归档到以该立意命名的文件夹，而非删除；可随时切回原立意恢复其剧本。**
> 关联 [[gui-web-rewrite-strategy]]、`项目文件规范.md`。

## 1. 现状与问题
- 当前 `/ideate/select` 在 `selected_id` 改变时调 `purge_downstream(stage="script_outline")` —— **直接删除** 剧本_E*.md / 分镜_E*.json / prompts/ / video_prompts/ 等。
- 换一个立意 = 永久丢失上一个立意已写的全部剧本/分镜/出图prompt。无法回退。

## 2. 设计原则
1. **永不删除下游产物**：换立意时把"当前下游产物"整体**移动**到归档区，再（若新立意有归档）恢复新立意的产物。
2. **以立意命名归档**：归档目录 = `归档/<idea_id>__<安全标题>/`，一眼可辨属于哪个立意。
3. **可逆**：切回旧立意即把其归档产物移回项目根，继续编辑。
4. **manifest 记账**：复用 `project.json.archive[]` 记录归档历史（已有该字段）。
5. **原子/安全**：移动用 `shutil.move`，目标已存在先腾挪；失败不留半态。

## 3. 目录与命名
```
<项目>/
  创意.json                      # 含 candidates[] / selected_id（当前生效立意）
  剧本_E1.md … 分镜_E1.json …     # 当前生效立意的下游产物（项目根）
  prompts/  video_prompts/ …      # 当前生效立意的下游
  归档/
    c1__替嫁后她惊艳全场/          # 立意 c1 的下游快照（被换下时归档于此）
      剧本_E1.md  分镜_E1.json  prompts/ …
    c3__重生归来复仇/             # 立意 c3 的下游快照
      …
```
- `<idea_id>` = 创意.json candidate 的 id（如 `c1`）。
- `<安全标题>` = candidate.title 经 `safe_name()`（去非法字符/截断 ~40 字）。
- 归档范围（"下游产物"白名单）：`剧本.json`、`剧本_E*.md`、`剧本.md`(旧)、`分镜_E*.json`、`分镜.json`(旧)、`prompts/`、`video_prompts/`、`audio_prompts/`。**不含** 创意.json（立意本身常驻）、`imggen/`/`video/`/`soundtrack/`（产物按需另议，默认不动以免误搬大文件——见 §6 开关）。

## 4. 切换流程（/ideate/select 改造）
当 `POST /ideate/select {project_dir, selected_id}` 且 `selected_id != 旧 selected_id`：
1. **归档旧立意下游**：若项目根存在下游白名单文件且旧 `selected_id` 非空 → 移动到 `归档/<旧idea>__<旧标题>/`（目标已存在则先并入/覆盖同名）。manifest.archive 追加/更新一条 `{idea_id, title, archived_at, files:[…]}`。
2. **写新 selected_id** 到 创意.json。
3. **恢复新立意下游（若有归档）**：若 `归档/<新idea>__<新标题>/` 存在 → 把其中产物移回项目根（恢复编辑），并从 manifest.archive 标记 active（或移除该归档条目）。无归档则项目根保持空（全新立意，待生成）。
4. 返回结构见 §5。
- `selected_id` **未变** → 不归档不恢复（沿用现有"同立意不清"的保护）。

## 5. 接口标准

### 5.1 POST /ideate/select（改造）
请求：`{project_dir: str, selected_id: str}`
响应：
```json
{
  "saved": "<创意.json 路径>",
  "selected": { "id": "c3", "title": "…" },
  "selection_changed": true,
  "archived": { "idea_id": "c1", "title": "替嫁…", "dir": "归档/c1__替嫁…", "files": ["剧本_E1.md","分镜_E1.json","prompts/"] },
  "restored": { "idea_id": "c3", "title": "重生…", "files": ["剧本_E1.md"] }
}
```
- `archived`/`restored` 为 null 表示该步未发生。**绝不返回 purged/删除**。

### 5.2 GET /project/archives?project=&lt;dir&gt;
列出所有归档立意（供 UI「往期立意」列表）：
```json
{ "archives": [
  { "idea_id":"c1", "title":"替嫁…", "dir":"归档/c1__替嫁…", "archived_at":"2026-06-01T13:30:00", "file_count": 5, "is_active": false },
  { "idea_id":"c3", "title":"重生…", "dir":"归档/c3__重生…", "archived_at":"…", "file_count": 1, "is_active": true }
]}
```

### 5.3 POST /project/archive/restore（显式切回，可选；端点用 ASCII，落盘目录仍是 归档/）
`{project_dir, idea_id}` → 把当前下游归档（按当前 selected_id）后，恢复指定 idea_id 的归档产物 + 把 创意.json.selected_id 设为该 idea。响应同 §5.1 的 archived/restored。
（注：§5.1 的 /ideate/select 已隐式做"归档+恢复"，本端点供"立意页往期列表点击直接切回"的显式入口。）

### 5.4 manifest.archive 记账格式
`project.json.archive`（已有字段）：
```json
[ { "idea_id":"c1", "title":"替嫁…", "dir":"归档/c1__替嫁…", "archived_at":"…", "files":["剧本_E1.md","分镜_E1.json","prompts/"] } ]
```

## 6. 配置开关
- `cfg.idea_switch_archive_enabled: bool = true` —— 关则回退旧的 purge 行为（不建议）。
- `cfg.idea_archive_include_media: bool = false` —— 是否把 imggen/video/soundtrack 大产物也纳入归档（默认否，避免搬运大文件；按需开）。

## 7. 前端配合（立意页）
- 立意页加「往期立意」入口：GET /project/archives 列出 → 点某条 = POST /project/archive/restore（或直接 select 该 idea_id）→ 切回其剧本。
- 推进/换立意时 toast 提示"已归档上一个立意《X》的剧本到 归档/，可在往期立意切回"，消除"误删"恐慌。
