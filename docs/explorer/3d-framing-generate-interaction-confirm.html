叉需要你拍板的设计分叉

① prompt 来源
  • 用户输入 + LLM 预填（推荐）：按框选区间的字幕/画面让 LLM 推荐一句，预填进输入框，可改
  • 纯用户输入：输入框默认空，最简，不接 LLM
LLM 预填更智能但要接一次推理调用；纯输入实现快。
② 异步生成的反馈方式
  • lane 内「生成中」占位块（推荐）：最符合 DAW 直觉，能多条并行生成
  • modal 进度条：阻塞，一次只能等一个
  • 角落 toast：不占轨，但定位感弱
推荐需给 OverlaySegment 加 status(pending/generating/generated/failed)。
③ 时长
  • 自动 = 框选长度（推荐）：t_end−t_start，所见即所得
  • 对话框可微调：多一个时长输入，灵活但啰嗦
BGM 走 compose_acestep_inputs(duration)；SFX 限 1–15s（管线约束）。
④ OverlaySegment.status 字段（小改 3a）
  • 加 status（推荐）：支持生成中/失败态、重试、占位渲染
  • 不加：audio_path 空即「未生成」，无法区分「生成中」与「失败」
加字段需同步 OverlaySegment.to_dict/from_dict + 一条迁移兼容（旧数据默认 generated）。

看完告诉我：① prompt 来源 / ② 反馈方式 / ③ 时长 / ④ 要不要加 status —— 或直接说「用推荐默认」。 其余已锁：框选=时间轴 Shift 拖拽、生成走 RunningHub 异步后台线程、落库=OverlaySession.add+save_overlay+引擎刷新。 （注：3d 实现要等 3c 工作流落地后再开。）
