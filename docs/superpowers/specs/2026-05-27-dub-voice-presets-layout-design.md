# 配音快捷音色提示词 + 界面重排 — 设计

**日期**：2026-05-27
**目标**：给配音「音色设计」加分类快捷提示词按钮（方言/人物身份/场合/情感语调），右栏分组可折叠；把配音任务窗尺寸与布局对齐视频生成窗（1100×820，左右两栏、更舒展）；底部增加「打开输出文件夹」。

## 研究依据（Qwen3-TTS，已确认）

- **方言靠"预置音色 voice 参数"，不靠音色描述文字**；官方支持 8 种（北京/上海/四川/粤/南京/陕西/天津/闽南）；**湖南话/长沙话官方未列出**。
- **音色描述(instruct)** 只支持中/英文，覆盖维度：性别/年龄/音色质感/语速/语调/情感/场景/身份；要具体、多维叠加、客观描述、勿模仿真人。
- **决策（用户拍板）**：方言按钮**照插描述文字**（含湖南话），但文档/提示注明"方言靠描述不一定生效"。身份/场合/情感类按钮靠音色描述真实生效。

## A. 快捷提示词数据（可外部配置加载）

**从 YAML 配置加载**，便于开发者更新最佳实践而无需改代码（项目已用 pyyaml，如 `ltx_v3_extras.yaml`，故用 YAML 一致）。

- **配置文件** `drama_shot_master/assets/voice_presets.yaml`（随 Nuitka `assets/` 一并发布，开发者直接编辑更新）。结构：
  ```yaml
  dialect_note: "提示：方言靠音色描述不一定生效（Qwen3-TTS 方言走预置音色）；湖南话官方未列出。"
  categories:
    - name: 方言
      items:
        - {label: 北京话, text: "用北京话朗读，京味儿胡同腔，自然地道，"}
        - {label: 上海话, text: "用上海话朗读，吴侬软语，语调婉转，"}
        # …（下方完整词条全部写入此 yaml）
    - name: 人物身份
      items: [...]
    - name: 场合
      items: [...]
    - name: 情感语调
      items: [...]
  ```
- **模块** `drama_shot_master/core/voice_presets.py`：
  - `_DEFAULTS`：内置同样的分类词条（作为兜底）+ `_DEFAULT_NOTE`。
  - `load_presets(path: Path | None = None) -> tuple[list, str]`：`path` 默认指向 `assets/voice_presets.yaml`（测试可传临时路径）；解析成功返回 `(categories, dialect_note)`（`categories` = `[(name, [(label,text),...]), ...]`）；文件缺失/解析失败/结构非法 → 返回内置 `_DEFAULTS`（不崩）。
  - 校验：每 item 必须有非空 `label`/`text`，否则跳过该条；分类为空则跳过。

## B. 界面重排

### 任务窗尺寸
`DubTaskWindow.resize(620, 720)` → `resize(1100, 820)`（与 `VideoTaskWindow`/`SoundtrackTaskWindow` 一致）。

### DubPanel 重排（左右两栏）
顶部仍是「音色设计 / 声音克隆」单选。下面主体改为 `QHBoxLayout` 两栏：

- **左栏（输入，stretch=3）**：现有的 `QStackedWidget`（音色设计页 / 声音克隆页）。各页排得更舒展（表单行距、文本框高度适当加大）。
- **右栏（快捷词面板，固定宽 ~300，stretch=0）**：`快捷音色提示词` 面板——按 `load_presets()` 返回的分类，每个分类一个**可折叠分组**：
  - 折叠头：`QToolButton`（checkable，箭头▸/▾ + 分类名），点击切换该组按钮区显隐。
  - 组内容：流式排列的小按钮（每个词条一个），点击把短语插入**目标文本框光标处**。
  - 顶部显示 `load_presets()` 返回的方言提示（灰色小字）。
  - 整列放进 `QScrollArea`（竖向滚动），保证分组多/小屏不挤。

底部操作条（横跨两栏下方）：`生成` | `打开结果` | **`打开输出文件夹`** | 状态标签。

布局示意：
```
( ⊙ 音色设计   ○ 声音克隆 )
┌────────左栏 输入(stretch3)────────┬───右栏 快捷词(≈300)───┐
│  音色设计页:                      │ [方言 ▾]            │
│   要合成文本 [__________]         │  北京 上海 粤语 …     │
│   音色描述   [__________]         │ [人物身份 ▸]         │
│   语言       [自动▾]              │ [场合 ▸]             │
│  声音克隆页: 文本/参考音频/        │ [情感语调 ▾]         │
│   情感强度/4选1情感模式…           │  温柔 激昂 悲伤 …     │
│                                   │ (灰: 方言提示)        │
└───────────────────────────────────┴──────────────────────┘
[生成] [打开结果] [打开输出文件夹]        状态…
```

### 插入目标（点快捷按钮插到哪个文本框）
`_insert_preset(text)`：
- 音色设计模式 → 插入 `d_style`（音色描述）光标处。
- 声音克隆模式 → 插入 `c_emo_text`（情感描述）光标处（情感类对"文本情绪"子模式可用；其它类插进去无害，由用户取舍）。

### 打开输出文件夹
新增 `btn_open_dir`：打开 `cfg.dub_output_dir`（空则当前目录）下的配音输出目录（与生成落盘目录一致）。用 `QDesktopServices.openUrl(QUrl.fromLocalFile(dir))`；目录不存在则先 `mkdir` 或提示。

### 可折叠分组组件
小助手 `_CollapsibleGroup(QWidget)`：构造 `(title, buttons_container)`；头 `QToolButton`(checkable, `setArrowType`/文本箭头)，`toggled` → `content.setVisible`。默认「方言」「情感语调」展开，「人物身份」「场合」折叠（减少初始高度）。

## C. 落地单元

1. `drama_shot_master/assets/voice_presets.yaml`（可编辑配置，全部词条）。
2. `core/voice_presets.py`（`_DEFAULTS` 兜底 + `load_presets()` 读 YAML）+ 单测。
3. `ui/panels/dub_panel.py`：左右两栏重排 + 右栏可折叠快捷词面板（数据来自 `load_presets()`）+ `_insert_preset` + `btn_open_dir`（打开输出文件夹）+ `QScrollArea` 兜底。保持原有信号/持久化/生成逻辑不变（快捷词插入会触发已接的 textChanged→dirty，自动存）。
4. `ui/windows/dub_task_window.py`：`resize(1100, 820)`。

## 测试

- `voice_presets`：`load_presets()` 返回 4 个分类、各分类词条数（方言9/身份8/场合6/情感8）、每条 (label,text) 且 text 非空、方言含「湖南话」、返回方言提示非空；
  **配置加载**：临时写一份精简 YAML → `load_presets` 解析出自定义分类；YAML 缺失/解析失败/结构非法 → 回退 `_DEFAULTS`（不崩）。
- UI 手动/离屏：DubPanel 构造、折叠分组展开/折叠、点按钮插入到音色描述/情感描述、`btn_open_dir` 存在、窗口 1100×820。

## 非目标

- 不改 TTS 提交/工作流逻辑、不加方言"预置音色"参数（待用户提供方言工作流再做真方言）。
- 不做按钮自定义编辑 UI（词条写死在 voice_presets.py，后续可扩展）。
